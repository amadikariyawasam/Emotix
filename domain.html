Research Domain

Project Domain

Literature survey, research gap, objectives, methodology and technologies.

01

Literature Survey

Previous research on workplace wellness relied on manual self-reporting, wearable devices, cloud-based monitoring, and basic attendance tracking systems. Traditional platforms used facial emotion recognition for stress detection but raised privacy concerns due to cloud processing. Recent studies explored computer vision for gaze tracking, deep learning for analyzing facial expressions, and edge computing for privacy. However, existing systems lack integration of routine learning, focus detection, stress analysis, and AI-based employee assistance in a single secure platform.

Studies on eye-tracking systems demonstrated effectiveness in measuring cognitive load, while temporal models such as LSTM networks showed promise for analyzing emotional trajectories over extended work sessions. Cybersecurity frameworks for IoT-based monitoring remain underexplored, especially in the context of sensitive biometric data collected in workplace environments.

02

Research Gap

Existing systems lack real-time mental wellness monitoring with privacy-preserving local processing. Current platforms do not combine adaptive routine learning, attention tracking, temporal stress detection, and conversational AI support in one ecosystem while ensuring data security and employee privacy.

No existing solution addresses the complete pipeline — from on-device biometric capture to encrypted storage, real-time analysis, and proactive conversational recommendations — within a single unified, deployable framework suitable for enterprise environments.

03

Research Problem

Modern workplaces create mental exhaustion and decreased focus through constant digital connectivity. Main impact areas include:

  • Privacy violations from cloud-based data exposure of sensitive biometric information.
  • Incomplete assessment — tracking only presence, not actual mental states or wellness.
  • Security gaps in monitoring infrastructure exposing employee data to cyber threats.
  • Lack of real-time support with delayed or generic wellness recommendations.
  • Employee distrust from invasive monitoring without adequate data protection.
04

Research Objectives

  • Develop an edge-based routine monitoring system for privacy-preserving individual work pattern learning using TinyML on Raspberry Pi.
  • Develop an eye-tracking system for real-time focus and fatigue detection using EAR algorithm, gaze estimation, and fixation tracking.
  • Develop a facial expression analysis system for temporal stress recognition using CNN and LSTM models with the FER+ dataset.
  • Develop an AI chatbot for personalized wellness recommendations based on aggregated focus, stress, and behavioral data.
  • Develop an encryption and cybersecurity framework (AES-256-GCM, HMAC-SHA256, AI Honeypot) for robust data protection.
05

Methodology

The EMOTIX pipeline is structured as a sequential, modular workflow from raw sensor data to actionable wellness insights — all processed locally on-device:

📷
Data Collection
Cameras capture facial expressions & eye movements; sensors record workplace conditions.
⚡
Edge Processing
Raspberry Pi processes data locally to learn work patterns via TinyML.
👁️
Computer Vision
Eye tracking computes focus scores from gaze, blink rate, and fixation data.
🧠
Deep Learning
CNN extracts emotions; LSTM analyzes temporal patterns for stress scoring.
💬
AI Chatbot
Converts analytics into personalized wellness recommendations.
🔒
Security
AES-256-GCM encrypts data; AI Honeypot defends infrastructure.
📊
Output
Dashboard displays real-time focus, stress, and wellness insights.
06

Technologies Used

🖥️ Hardware

  • Raspberry Pi (edge computing)
  • Camera sensors (facial & eye)
  • Environmental sensors
  • Light, temperature, noise

🤖 ML & AI

  • TinyML (routine learning)
  • CNN (facial emotion recognition)
  • LSTM (temporal analysis)
  • Isolation Forest (anomaly detection)

👁️ Computer Vision

  • OpenCV (landmark detection)
  • MediaPipe FaceMesh
  • EAR Algorithm (blink detection)
  • Gaze & fixation tracking

🔐 Security

  • AES-256-GCM encryption
  • HMAC-SHA256 integrity
  • AI-based Honeypot
  • HTTP & MQTT monitoring

📐 Models & Frameworks

  • FER+ Dataset
  • Russell Circumplex Model
  • Rule-based AI chatbot
  • Weighted scoring algorithms

🚀 Deployment

  • Web-based dashboard
  • Real-time monitoring UI
  • Local edge architecture
